# 社区行为准则

WutherCore 希望为使用者和贡献者提供一个专业、友善、能够就技术问题坦诚讨论的空间。

## 我们期待的行为

- 尊重不同经验、背景和观点；
- 针对代码、设计和事实讨论，不攻击个人；
- 给出可执行、具体的反馈，并愿意解释判断依据；
- 承认错误，及时修正；
- 在日志、配置和截图中保护他人的隐私与凭据；
- 关注项目和社区的长期利益。

## 不可接受的行为

- 骚扰、威胁、歧视、侮辱或持续的人身攻击；
- 未经允许公开他人的身份、联系方式、网络信息或私密交流；
- 发布代理凭据、访问令牌、私钥或其他敏感数据；
- 垃圾信息、恶意破坏、蓄意误导或反复打断正常协作；
- 在项目空间内进行其他明显不专业或不安全的行为。

## 适用范围

本准则适用于仓库的 Issue、Discussion、Pull Request、Review、提交记录和与项目直接相关的公开交流。代表项目参与其他活动时同样适用。

## 执行

维护者可以编辑或删除违规内容，关闭或锁定讨论，拒绝贡献，并在严重或反复违规时限制参与。处理时会综合考虑行为影响、上下文、是否主动修正以及历史记录。

如需报告行为问题，请通过仓库拥有者的 GitHub 主页提供的私下联系方式联系。报告会在合理范围内保密；涉及安全漏洞时，请改用 [SECURITY.md](SECURITY.md) 中的私密报告渠道。
